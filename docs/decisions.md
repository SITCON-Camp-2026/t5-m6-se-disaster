# 需求取捨決策

> 這份文件記錄 Phase 0 / Release 01 的取捨和判斷。

## 我決定 v1 優先服務的使用者

- [x] 資訊整理者

原因：v1 的原型應該先保留來源與查核狀態，避免讓行動者誤信未確認資訊。

## 這個使用者最想完成什麼

- 先把混亂的原始回報整理成下一位協作者能理解的候選資料。
- 保留原始來源、查核狀態，以及哪些資訊需要再確認。

## 我今天暫時不處理的需求

- 不處理直接給行動者「可行動 / 不可行動」的明確判斷。
- 不處理把社群貼文直接轉成正式任務。

## 我不採用的 AI 建議

- AI 建議：把所有 `sourceType` 為「社群貼文」的資料自動轉成任務候選。
  - 不採用原因：來源未經查核，這樣會增加誤導風險。

## 必須保留人工確認的判斷

- `needs_review` 和 `unverified` 的資料不能直接當作已確認資訊。
- 應該在 UI 中清楚標示原始資料與整理者判斷之間的差別。

## 今天明確不做什麼

- 不做正式任務分派功能。
- 不做新增地圖、外部資料或真實 API 串接。

## 我仍不確定的地方

- 是否要在 v1 中加入「可行動 / 不可行動」這類建議欄位。
- 如何同時讓回報者、整理者、行動者都理解畫面中的查核狀態。
